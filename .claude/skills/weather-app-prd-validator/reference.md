# Reference（口径与资料索引）

本文件用于收纳"天气 PRD 校验"所需的**资料路径与口径约束**，避免把绝对路径与长清单塞进 `SKILL.md`。

## 1) 直接复用的口径与模版索引（首选）

优先阅读并引用（避免重复维护）：
- `../mobile-weather-app-prd/SKILL.md` - PRD 生成技能，包含完整模版结构
- `../system-weather-competitive-analysis/SKILL.md` - 竞品分析技能

这些文件中包含：
- PRD 输出模版路径与章节结构
- 用户洞察调研与提取方式
- 生活指数口径（华风/墨迹）与"等级映射/归一化"约束
- 预警类型枚举（权威枚举与字段口径）
- 竞品分析强制范围（小米/华为/荣耀/墨迹）

## 2) 校验时建议额外盯紧的"天气特有口径"

### 时间与日界线
- "今日/今晚/明日"的划分是否按**城市本地时区**计算
- 日出日落、逐小时曲线的时间轴与夏令时（如适用）
- 跨日界线（如 23:00 - 02:00）的处理规则

### 单位与显示
- 温度：℃/℉，精度（整数/小数）
- 风力：m/s、km/h、级数、阵风标注
- 能见度：m/km，切换阈值
- 降水：mm/小时、概率 %，精度要求
- AQI：等级阈值（国标/地方标准差异）

### 刷新与缓存（最低需要明确的 6 个点）
1. **前台进入/下拉刷新触发**
   - 触发条件（进入前台/下拉手势/定时轮询）
   - 最小刷新间隔（防止频繁请求）
2. **后台刷新/系统调度触发**（如果做）
   - 调度策略（固定间隔/系统调度/条件触发）
   - 后台执行权限与限制
3. **数据失败重试与退避**
   - 重试次数与间隔
   - 指数退避策略
   - 失败后的 UI 状态
4. **缓存有效期与过期提示**
   - 各类数据的缓存时长
   - 过期后的展示策略
   - "上次更新时间"的展示规则
5. **离线展示策略**
   - 展示缓存数据的范围
   - 局部占位/置灰规则
   - 离线提示文案与位置
6. **限流/熔断/降级**
   - 服务端限流时的客户端处理
   - 核心字段 vs 非核心字段的降级策略
   - 熔断恢复机制

## 3) 评分卡打分建议（用于稳定输出）

统一用 0–5 分，建议口径（可按实际调整但要自洽）：

| 分数 | 含义 | 判定标准 |
|---:|---|---|
| 5 | 完美 | 信息完整且可直接开发/测试，几乎无 P0/P1 |
| 4 | 良好 | 少量 P1，边界与验收基本可落地 |
| 3 | 有缺口 | 有明显缺口（多处待确认），需要补齐后再开工 |
| 2 | 风险高 | 关键流程/口径/合规缺失，风险高 |
| 1 | 概念级 | 只有愿景/概念描述，几乎不可执行 |
| 0 | 不可用 | 与需求无关或信息严重不足 |

## 4) 预警类型与等级枚举（权威来源）

> ⚠️ 校验时如 PRD 未引用权威枚举，必须标记为"待确认"

### 气象预警颜色等级（国标）
| 颜色 | 等级 | 含义 |
|---|---|---|
| 蓝色 | IV 级（一般） | 可能发生一般气象灾害 |
| 黄色 | III 级（较重） | 可能发生较重气象灾害 |
| 橙色 | II 级（严重） | 可能发生严重气象灾害 |
| 红色 | I 级（特别严重） | 可能发生特别严重气象灾害 |

### 常见气象预警类型
- 暴雨、暴雪、寒潮、大风、沙尘暴、高温
- 干旱、雷电、冰雹、霜冻、大雾、霾
- 道路结冰、雷雨大风、森林火险、台风

## 5) 生活指数口径参考

### 华风生活指数（5 级制）
| 等级 | 数值范围 | 含义示例（以穿衣指数为例） |
|---:|---|---|
| 1 | 0-20 | 非常舒适，单衣即可 |
| 2 | 21-40 | 较舒适，轻薄外套 |
| 3 | 41-60 | 一般，需要外套 |
| 4 | 61-80 | 较差，需要厚外套 |
| 5 | 81-100 | 很差，需要棉衣 |

### 墨迹生活指数（多档制）
墨迹部分指数采用不同档位，校验时需确认：
- 是否有等级归一化方案（映射到 1-5）
- 阈值触发规则是否基于归一化后的等级
- 不同指数的档位差异是否已处理

### 常见生活指数类型
- 穿衣指数、带伞指数、紫外线指数
- 运动指数、洗车指数、钓鱼指数
- 晾晒指数、感冒指数、护肤指数
- 空气污染扩散指数、过敏指数

## 6) 定位与权限相关口径

### 定位精度等级
| 精度 | 适用场景 | 耗电影响 |
|---|---|---|
| 精确定位（GPS） | 分钟级降雨、精准预警 | 高 |
| 粗略定位（网络/基站） | 城市级天气 | 低 |
| 无定位 | 手动选城 | 无 |

### 权限状态处理
| 状态 | 处理策略 |
|---|---|
| 已授权 | 正常获取定位，展示定位城市天气 |
| 仅一次 | 本次获取后提示用户，下次需重新授权 |
| 拒绝 | 展示手动选城入口，引导去设置开启 |
| 系统关闭 | 展示系统设置入口引导 |
| 定位超时 | 使用缓存定位或提示手动选城 |

## 7) AQI 口径参考（国标 HJ 633-2012）

| AQI 范围 | 等级 | 类别 | 颜色 |
|---:|---|---|---|
| 0-50 | 一级 | 优 | 绿色 |
| 51-100 | 二级 | 良 | 黄色 |
| 101-150 | 三级 | 轻度污染 | 橙色 |
| 151-200 | 四级 | 中度污染 | 红色 |
| 201-300 | 五级 | 重度污染 | 紫色 |
| >300 | 六级 | 严重污染 | 褐红色 |

### 主要污染物
- PM2.5（细颗粒物）
- PM10（可吸入颗粒物）
- O3（臭氧）
- NO2（二氧化氮）
- SO2（二氧化硫）
- CO（一氧化碳）

## 8) 竞品对标范围（强制覆盖）

校验时需确认 PRD 是否参考了以下竞品：
1. **小米天气** - MIUI 系统内置
2. **华为天气** - 鸿蒙/EMUI 系统内置
3. **荣耀天气** - MagicOS 系统内置
4. **墨迹天气** - 第三方头部应用

### 对标维度建议
- 核心功能覆盖度
- 数据源与准确性策略
- 刷新与缓存机制
- 预警展示与通知策略
- 生活指数种类与展示
- 桌面组件/负一屏能力
- 性能与流畅度表现

## 9) 合规检查要点

### 隐私合规（必查项）
- [ ] 定位权限：采集目的、精度、频率、最小化原则
- [ ] 推送权限：通知内容、频率、免打扰、退订方式
- [ ] 设备信息：采集范围、目的、脱敏处理
- [ ] 第三方 SDK：清单、数据共享说明

### 进网要求（如适用）
- [ ] 数据存储位置（境内/境外）
- [ ] 数据传输加密
- [ ] 日志保留周期
- [ ] 用户数据导出/删除能力

## 10) 常见校验遗漏点速查表

| 遗漏类型 | 典型表现 | 校验建议 |
|---|---|---|
| 口径模糊 | "适当提示"、"合理展示" | 追问：何时触发？展示在哪？文案是什么？ |
| 缺少兜底 | 只写正常流程 | 追问：失败/空/超时怎么办？ |
| 验收不可测 | "更好/更准/更流畅" | 追问：怎么量化？测试用例是什么？ |
| 埋点缺失 | MOS 有指标但无事件 | 追问：数据从哪来？采集口径是什么？ |
| 合规遗漏 | 涉及权限但 6.x 未填 | 追问：目的？最小化？拒绝路径？ |
| 策略冲突 | 多处定义不一致 | 全篇搜索同一概念的不同表述 |
